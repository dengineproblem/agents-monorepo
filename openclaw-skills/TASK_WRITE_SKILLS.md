# Задание: Создать скиллы для платформы Performante

## Твоя задача

Ты — OpenClaw, AI-агент на сервере платформы Performante. Твоя цель — стать полноценным коммерческим агентом, который управляет рекламой, CRM, ботами и аналитикой.

Для этого тебе нужно создать набор скиллов (SKILL.md файлов), используя:
1. **PROJECT_REFERENCE.md** — полный справочник платформы (API, бизнес-логика, БД)
2. **~/project/** — клон исходного кода проекта (читай код для деталей)
3. **БД** — read-only доступ через SQL для проверки данных

## Как работать

### Порядок создания (по приоритету)

1. **morning-briefing** — ежедневная сводка (лиды, реклама, продажи, проблемы)
2. **ads-dashboard** — метрики рекламы, CPL, ROI, Health Score
3. **crm-leads** — управление лидами, воронка, распределение
4. **ads-optimization** — автопилот рекламы (пауза, бюджеты, ad-eater)
5. **alerting** — проактивный мониторинг и алерты
6. **campaign-builder** — создание кампаний и тестов креативов
7. **crm-consultants** — KPI консультантов, расписание, звонки
8. **chatbot-management** — промпты ботов, кампании рассылок
9. **creative-generation** — генерация текстов и изображений
10. **competitor-intelligence** — мониторинг конкурентов
11. **analytics-reporting** — отчёты за период
12. **integrations** — AmoCRM, Bitrix24, CAPI

### Процесс для каждого скилла

1. Изучи соответствующие API endpoints из PROJECT_REFERENCE.md (секция 2)
2. Изучи бизнес-логику (секция 3) и схему БД (секция 4)
3. При необходимости загляни в код: `~/project/services/{service}/src/`
4. Напиши SKILL.md по формату из PROJECT_REFERENCE.md (секция 5.1)
5. Сохрани в `skills/{skill-name}/SKILL.md`
6. **Обсуди с пользователем** — покажи что получилось, спроси фидбек
7. Исправь по замечаниям
8. Переходи к следующему скиллу

### Формат SKILL.md

Смотри PROJECT_REFERENCE.md секция 5.1. Ключевое:
- YAML frontmatter: name, description, requires
- READ и WRITE инструменты с curl примерами
- Все вызовы через localhost (Docker сеть)
- Авторизация: `x-user-id: e1a3a32a-d141-407c-b92e-846e5869f63d`
- Русский язык
- Формат ответов для Telegram (эмодзи, компактные карточки)

### Что важно

- **Не выдумывай endpoints** — используй только те, что описаны в PROJECT_REFERENCE.md
- **Проверяй через SQL** — если не уверен в структуре данных, сделай `\d+ table_name`
- **Читай код** — если нужны детали, загляни в `~/project/services/`
- **Обсуждай** — после каждого скилла спроси пользователя, всё ли правильно
- **Итерируй** — лучше 1 хороший скилл, чем 12 поверхностных

## Начни

Прочитай PROJECT_REFERENCE.md целиком. Потом начни с **morning-briefing** — это самый показательный скилл, потому что использует данные из всех сервисов (реклама, лиды, CRM, боты).
